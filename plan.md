# 毕业设计执行计划

## 课题：面向科学场景的智能体工具调用优化研究

---

## 与开题报告的主要调整

- **砍掉 Biopython 生物信息学工具**：四大类缩减为三大类（数值计算、统计分析、数据可视化），论文中提"可扩展"即可
- **简化动态工具选择策略**：依靠 LLM 自身推理选择工具，不单独写启发式引擎
- **简化跨工具数据流管理**：用 Python dict/JSON 做中间传递，不做重量级数据流规范
- **简化工具白名单与参数校验**：做基本的参数类型检查 + 工具名校验
- **对比实验调整**：原生 LLM 直接调用 vs 本系统优化后的效果
- **测试用例**：从 20 个缩减到 10-12 个，保证覆盖主要类别
- **LLM 选型**：建议用通义千问 API（国内直接访问、有免费额度、支持 Function Calling），论文中写"基于 OpenAI 兼容接口"

## 核心保留内容

1. 工具抽象基类 + 三大类工具封装
2. 基于 Prompt Engineering 的任务拆解机制（论文核心亮点）
3. 基本的容错重试 + 结果校验
4. 端到端可运行的系统原型
5. 对比实验（证明优化有效）

## 执行步骤

### 阶段一：环境搭建与项目基建

1. 搭建开发环境：Python 虚拟环境、Git 仓库、跨平台配置（换行符、编码等）
2. 确定项目目录结构，创建骨架代码
3. 配置 LLM API 连接（申请通义千问 API Key，验证可用性）

### 阶段二：科学工具库开发

4. 设计并实现工具抽象基类 `BaseTool`
5. 封装数值计算类工具（3-4 个）：矩阵运算、数值积分、函数拟合、方程求解
6. 封装统计分析类工具（3-4 个）：描述性统计、假设检验、回归分析、相关性分析
7. 封装数据可视化类工具（3-4 个）：折线图、散点图、柱状图、热力图
8. 为所有工具编写单元测试

### 阶段三：Agent 核心框架搭建

9. 搭建 LLM 调用封装层（支持 OpenAI 兼容接口）
10. 实现工具注册与管理机制
11. 实现 ReAct 循环核心逻辑（思考 → 行动 → 观察 → 循环）
12. 端到端联调：用户输入 → Agent 思考 → 工具调用 → 结果返回

### 阶段四：Prompt 工程与任务拆解优化

13. 设计系统级提示词（角色定义、行为规范、输出格式）
14. 设计科学任务拆解提示词模板
15. 编写 few-shot 示例（覆盖不同类型的科研任务）
16. 多轮测试与 Prompt 迭代优化

### 阶段五：容错机制与结果校验

17. 实现工具调用失败自动重试机制
18. 实现基本的参数校验（类型检查、范围检查）
19. 实现结果合理性检查（数值范围、统计显著性等）

### 阶段六：实验与验证

20. 构建测试数据集（10-12 个不同类型的科学任务）
21. 设计对比实验方案（原生 LLM vs 本系统）
22. 执行实验，记录指标（任务成功率、工具调用准确率、平均执行步数）
23. 选取 2-3 个真实科研场景做可用性验证
24. 整理实验数据，制作图表

### 阶段七：论文撰写与收尾

25. 撰写论文：背景与意义、相关工作综述、系统设计、核心实现、实验分析、结论
26. 论文修改与格式调整
27. 整理代码仓库、编写 README 和使用说明
28. 准备答辩 PPT

## 技术栈

| 组件 | 选型 |
|------|------|
| 编程语言 | Python 3.10+ |
| LLM API | 通义千问 (qwen-plus/qwen-max)，OpenAI 兼容接口 |
| Agent 框架 | LangChain（轻量使用）或手写 ReAct 循环 |
| 数值计算 | NumPy, SciPy |
| 统计分析 | Pandas, SciPy.stats |
| 数据可视化 | Matplotlib, Seaborn |
| 测试 | pytest |
| 环境管理 | venv + requirements.txt |
| 版本控制 | Git |

## 跨平台注意事项

- Git 配置 `core.autocrlf` 和 `.gitattributes` 处理换行符
- 使用 `.env` 文件管理 API Key，加入 `.gitignore`
- 路径一律使用 `pathlib.Path` 避免 `/` 和 `\` 问题
- 统一 Python 版本（两台机器保持一致）
